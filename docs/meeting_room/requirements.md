## 1. プロジェクト概要

### 目的
社内の会議室（A, B, C）を効率的に予約・管理するためのWebアプリケーション。

### 対象ユーザー
- 社内社員（Googleアカウント保有者）

### 技術スタック
- **Frontend**: Next.js (App Router)
- **Backend**: Supabase (Auth, DB)
- **Styling**: Tailwind CSS
- **認証**: Google OAuth via Supabase Auth

### 会議室情報
| ID | 名称 | 収容人数 |
|----|------|----------|
| room_a | 会議室A | 10人 |
| room_b | 会議室B | 6人 |
| room_c | 会議室C | 4人 |

## 2. ユーザーストーリー

### US-001: Googleアカウントログイン

**ストーリー**
As a **社員**,
I want **Googleアカウントでログインできる**,
So that **会議室予約システムに安全にアクセスできる**.

**受け入れ条件**
- [ ] **Given**: 未ログイン状態のユーザー
      **When**: トップページにアクセス
      **Then**: Googleログインボタンが表示される
- [ ] **Given**: Googleログインボタンをクリック
      **When**: Google認証画面で認証完了
      **Then**: カレンダー画面にリダイレクトされる
- [ ] **Given**: 認証に失敗した場合
      **When**: エラーが発生
      **Then**: エラーメッセージが表示され、再試行が可能

**優先度**: **Must** (P1)
**見積もり**: 3ポイント

---

### US-002: 会議室空き状況確認

**ストーリー**
As a **社員**,
I want **会議室（A, B, C）の空き状況をカレンダーで確認できる**,
So that **予約可能な時間を一目で把握できる**.

**受け入れ条件**
- [ ] **Given**: ログイン済みユーザー
      **When**: カレンダー画面を表示
      **Then**: 3つの会議室の週間スケジュールが表示される
- [ ] **Given**: カレンダー上の予約済み時間
      **When**: 時間枠を確認
      **Then**: 予約者名と会議タイトルが表示される
- [ ] **Given**: カレンダー上の空き時間
      **When**: 時間枠を確認
      **Then**: 空きであることが視覚的に分かる（白または薄いグレー）
- [ ] **Given**: 収容人数の確認
      **When**: 会議室名にマウスオーバー
      **Then**: 収容人数が表示される

**優先度**: **Must** (P1)
**見積もり**: 5ポイント

---

### US-003: 会議室予約

**ストーリー**
As a **社員**,
I want **空いている時間をクリックして予約できる**,
So that **会議のために会議室を確保できる**.

**受け入れ条件**
- [ ] **Given**: ログイン済みユーザー
      **When**: カレンダー上の空き時間枠をクリック
      **Then**: 予約フォーム（モーダル）が表示される
- [ ] **Given**: 予約フォームに会議タイトル、開始時刻、終了時刻を入力
      **When**: 「予約する」ボタンをクリック
      **Then**: 予約が保存され、カレンダーに反映される
- [ ] **Given**: 予約可能時間は30分単位
      **When**: 時間を選択
      **Then**: 30分刻みで選択できる
- [ ] **Given**: 必須項目（会議タイトル、時刻）が未入力
      **When**: 「予約する」ボタンをクリック
      **Then**: バリデーションエラーメッセージが表示される
- [ ] **Given**: 既に予約済みの時間帯
      **When**: 重複する予約を試みる
      **Then**: エラーメッセージ「この時間は既に予約されています」が表示される

**優先度**: **Must** (P1)
**見積もり**: 5ポイント

---

### US-004: 予約一覧確認

**ストーリー**
As a **社員**,
I want **自分の予約一覧を確認できる**,
So that **今後の予約状況を把握できる**.

**受け入れ条件**
- [ ] **Given**: ログイン済みユーザー
      **When**: 「マイ予約」ボタンをクリック
      **Then**: 自分が作成した予約の一覧が表示される
- [ ] **Given**: 予約一覧
      **When**: 表示内容を確認
      **Then**: 会議室名、日時、会議タイトルが表示される
- [ ] **Given**: 過去の予約
      **When**: 一覧を表示
      **Then**: 過去の予約はグレーアウトまたは別セクションに表示される

**優先度**: **Should** (P2)
**見積もり**: 3ポイント

---

### US-005: 予約変更

**ストーリー**
As a **社員**,
I want **予約の日時や会議室を変更できる**,
So that **スケジュール変更に柔軟に対応できる**.

**受け入れ条件**
- [ ] **Given**: 自分の予約をクリック
      **When**: 「編集」ボタンをクリック
      **Then**: 予約編集フォーム（モーダル）が表示される
- [ ] **Given**: 予約編集フォームで日時または会議室を変更
      **When**: 「更新する」ボタンをクリック
      **Then**: 変更が保存され、カレンダーに反映される
- [ ] **Given**: 変更後の時間が既に予約済み
      **When**: 更新を試みる
      **Then**: エラーメッセージ「変更後の時間は既に予約されています」が表示される
- [ ] **Given**: 他人の予約
      **When**: 編集を試みる
      **Then**: 編集ボタンが表示されない（または無効化される）

**優先度**: **Should** (P2)
**見積もり**: 3ポイント

---

### US-006: 予約キャンセル

**ストーリー**
As a **社員**,
I want **予約をキャンセルできる**,
So that **不要になった予約を解除し、他の社員が利用できるようにする**.

**受け入れ条件**
- [ ] **Given**: 自分の予約をクリック
      **When**: 「キャンセル」ボタンをクリック
      **Then**: 確認ダイアログが表示される
- [ ] **Given**: 確認ダイアログで「はい」を選択
      **When**: キャンセル確定
      **Then**: 予約が削除され、カレンダーから消える
- [ ] **Given**: 他人の予約
      **When**: キャンセルを試みる
      **Then**: キャンセルボタンが表示されない（または無効化される）

**優先度**: **Must** (P1)
**見積もり**: 2ポイント

---

### US-007: 会議室情報表示

**ストーリー**
As a **社員**,
I want **会議室の収容人数を確認できる**,
So that **参加人数に適した会議室を選択できる**.

**受け入れ条件**
- [ ] **Given**: カレンダー画面
      **When**: 会議室名を表示
      **Then**: 会議室名の横に収容人数が表示される（例: 会議室A (10人)）
- [ ] **Given**: 予約フォーム
      **When**: 会議室を選択
      **Then**: 各選択肢に収容人数が表示される

**優先度**: **Could** (P3)
**見積もり**: 1ポイント

---

### US-008: ログアウト

**ストーリー**
As a **社員**,
I want **ログアウトできる**,
So that **セキュアにシステムを終了できる**.

**受け入れ条件**
- [ ] **Given**: ログイン済みユーザー
      **When**: 「ログアウト」ボタンをクリック
      **Then**: ログアウトが実行され、ログイン画面にリダイレクトされる
- [ ] **Given**: ログアウト後
      **When**: カレンダー画面に直接アクセスを試みる
      **Then**: ログイン画面にリダイレクトされる

**優先度**: **Must** (P1)
**見積もり**: 1ポイント

## 5. 技術的制約・前提条件

### UI/UXガイドライン
- シンプルで直感的なデザイン
- 青基調のカラースキーム
- レスポンシブ対応（デスクトップ優先、モバイルは後回し）

### セキュリティ要件
- Google OAuth認証必須
- 自分の予約のみ編集・削除可能
- ログイン必須（全画面）

### パフォーマンス要件
- カレンダー表示: 2秒以内
- 予約作成: 1秒以内
- リアルタイム更新不要（ページリロードでOK）

### データ要件
- 予約時間単位: 30分
- 営業時間: 9:00 - 18:00（暫定）
- タイムゾーン: Asia/Tokyo

## 6. スコープ外（今回のMVPでは実装しない）

| 項目 | 理由 |
|------|------|
| 繰り返し予約 | 複雑性を避けるため |
| メール通知 | インフラコスト削減 |
| 管理者機能 | MVPは利用者機能に集中 |
| カレンダー連携 | 外部API依存を避ける |
| 多言語対応 | 社内用（日本語のみ） |
| 会議室設備情報 | 最小機能に絞る |

## 8. Tech Leadへの引き継ぎ情報

### 実装推奨順序
1. **Phase 1**: 認証基盤（US-001, US-008）
2. **Phase 2**: カレンダー表示（US-002）
3. **Phase 3**: 予約作成（US-003）
4. **Phase 4**: 予約キャンセル（US-006）
5. **Phase 5**: マイ予約・変更（US-004, US-005）- Should扱い

### データベーススキーマ案
```sql
-- テーブル: bookings
CREATE TABLE bookings (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  room_id TEXT NOT NULL, -- 'room_a', 'room_b', 'room_c'
  title TEXT NOT NULL,
  start_time TIMESTAMPTZ NOT NULL,
  end_time TIMESTAMPTZ NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  CONSTRAINT no_overlap UNIQUE (room_id, start_time, end_time)
);
```

### 推奨ライブラリ
- カレンダーUI: `react-big-calendar` or `@fullcalendar/react`
- 日時処理: `date-fns`
- フォーム管理: `react-hook-form` + `zod`

### 注意事項
- 過剰な機能追加禁止（MVP範囲を厳守）
- UI/UX Designerからモックが提供される（別途）
- QA Reviewerによるテスト実施予定
